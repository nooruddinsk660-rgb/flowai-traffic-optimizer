## вҡЎ The Problem We Solve

> *"In India, an ambulance takes an average of **18вҖ“25 minutes** to travel 5 km in peak-hour Delhi traffic. Every red light is a potential life lost."*

Traditional traffic signals are **dumb timers**. They don't know an ambulance is 200m away. They don't know a junction is at 94% capacity. They can't coordinate a green wave through 4 intersections in 200ms.

**FlowAI changes all three вҖ” using only existing cameras and a laptop.**

## рҹӨ– AI Models

### Vision вҖ” YOLOv8n + Custom Ambulance Detector

| Metric | Value |
|--------|-------|
| Base model | YOLOv8n (pre-trained COCO) |
| Fine-tune dataset | ~800 labelled Indian ambulance images |
| **mAP50** | **96.7%** |
| **mAP50-95** | **88.6%** |
| Inference speed | ~12ms / frame (CPU) |
| Classes detected | car В· bus В· truck В· motorcycle В· person В· **ambulance** |

Training notebook: [`train_colab.ipynb`](https://github.com/nooruddinsk660-rgb/flowai-traffic-optimizer/blob/main/train_colab.ipynb)

### Audio вҖ” YAMNet (Google, 521 classes)

Siren detection uses class indices **397вҖ“400** (civil defense siren, ambulance, fire truck, police). Confidence is written to Redis with a **3s TTL** вҖ” stale audio never falsely holds a green light.

### Forecasting вҖ” XGBoost

```python
features = [hour, day_of_week, is_festival, is_peak,
            density_lag_1, density_lag_2, density_lag_3, aqi]
# Festival dates include: March 28, 2026 (Holi) вң“
# Output: 6 forecast slots вҖ” [t+5, t+10, t+15, t+20, t+25, t+30 min]
# RВІ вүҘ 0.85 on held-out test set
```

### Routing вҖ” NetworkX Dijkstra

7 pre-computed ambulance routes using real geodesic distances between 8 Delhi intersections. `DEMO_MODE=true` returns instant cached results; production runs live Dijkstra on the weighted graph.
